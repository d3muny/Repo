# Collaboration Global Rules

## 1. コードヘッダ規則
- すべての対象 `.cs` ファイル先頭コメントに以下を記載する。
  - コードの最終目的
  - バージョン名（`ver01` 開始、`ver02`...）
  - コンパイル不能修正は接尾語（`ver02a` など）
  - バージョン差分（1行）
  - バージョン更新日（分まで）

## 2. Git運用
- commit は原則 Codex が実施する。
- push はクライアントから明示指示があった時のみ実施する。

## 3. 応答運用
- 返答は会話番号を先頭に付ける（`会話NNN`）。
- 返答末尾に現在時刻を表示する。
- 23:30を超える場合は `夜更かし警告` を付与する。

## 4. アクセス範囲運用
- Codexの参照先は原則として以下に限定する。
  - 作業フォルダ（`Repo` 配下）
  - VRChatログファイル
- 上記以外へアクセスする場合は、Codexから事前に問い合わせを行い、クライアントの許可がある場合のみ実施する。
